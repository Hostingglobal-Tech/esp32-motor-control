# ESP32 Motor Control — WiFi + MCP 원격 모터 제어

![cover](docs/cover.png)

ESP32 자체에 HTTP 서버를 띄우고, 폰/PC/AI 어디서든 WiFi 로 모터를 제어한다.
AI(Claude 등)는 표준 MCP 프로토콜로, 사람은 브라우저/curl 로 — 두 경로 모두 ESP32 하나로 수렴.

```
Claude (MCP tools) ─→ esp32-motor-mcp (Rust stdio MCP) ─→ HTTP ─→ ESP32 웹서버 ─→ 연속회전 서보
폰 브라우저 / curl ──────────────────────────────────────→ ↗
```

- 펌웨어: Arduino (ESP32-S3 / 일반 ESP32), 외부 라이브러리 의존 0
- MCP 서버: Rust, JSON-RPC stdio 직접 구현 (의존성 serde_json 뿐)
- 범용 시퀀스 언어: "왼쪽 2바퀴 돌고 1초 쉬고 오른쪽 1바퀴" = `L2,W1,R1`

## 1. 준비 부품

| 부품 | 주문 검색어 (쿠팡/디바이스마트/엘레파츠) | 참고 |
|---|---|---|
| ESP32 개발보드 (검정, USB-C 2개) | **ESP32-S3-DevKitC-1** 또는 "ESP32-S3 개발보드 N16R8" | USB-C 1개짜리 일반 "ESP32 DevKit V1"(WROOM-32)도 동일 동작 |
| 파란색 소형 모터 (360도 회전) | **FS90R 연속회전 서보** 또는 "SG90 360도 연속회전 서보" | 일반 SG90(180도)은 바퀴 회전 불가 — 반드시 "360도/연속회전(continuous rotation)" 표기 확인 |
| 연결선 | 점퍼케이블 암-수 (F-M) 3개 | 서보 케이블 3핀 |

일반 SG90 = 각도 제어용(0~180도). "2바퀴 돌려" 같은 회전 명령을 쓰려면 연속회전형이어야 함.
정밀한 바퀴 수가 중요해지면 스텝모터(28BYJ-48 + ULN2003 드라이버)로 업그레이드 권장.

## 2. 배선

| 서보 선 | ESP32 핀 |
|---|---|
| 주황/노랑 (신호) | GPIO 4 |
| 빨강 (VCC) | 5V (VIN) |
| 갈색/검정 (GND) | GND |

USB 5V 전원으로 서보 1개는 충분. 여러 개 달면 외부 5V 전원 분리.

## 3. 펌웨어 업로드 (Arduino IDE)

1. Arduino IDE 설치 → 환경설정 → 추가 보드 매니저 URL:
   `https://espressif.github.io/arduino-esp32/package_esp32_index.json`
2. 보드 매니저에서 "esp32 by Espressif" 설치
3. 보드 선택: **ESP32S3 Dev Module** (일반 ESP32면 "ESP32 Dev Module")
4. 업로드 → 시리얼 모니터(115200)에 접속 SSID 와 IP 출력됨. 이후 USB 분리 가능(전원만 연결)

WiFi 연결 순서 (자동):
1. **NVS 에 저장된 공유기** (아래 등록 방법 참고 — 비밀번호는 소스코드에 넣지 않는다)
2. 공개(암호 없는) AP 스캔 → 신호 센 순으로 최대 5곳 자동 연결
3. 둘 다 실패 시 자체 AP(`ESP32-MOTOR` / `motor1234`) → 폰으로 접속 후 `http://192.168.4.1`

공유기 등록 (1회, 기기 NVS 에 저장 — 재부팅 후에도 유지):
- 방법 A (시리얼): 시리얼 모니터(115200)에 `WIFI <ssid> <비밀번호>` 입력 → 저장 후 자동 재부팅
- 방법 B (웹): AP 폴백 모드에서 `http://192.168.4.1` 접속 → "WiFi 공유기 등록" 폼 입력
- 방법 C (API): `curl "http://<IP>/api/wifi?ssid=<ssid>&pass=<pw>"`

주의: 공개 AP 는 암호화가 없는 망이다. 이 장치는 모터 명령만 주고받으므로 실용상 문제 없지만, 상시 운용은 본인 공유기 등록을 권장.

## 4. 사용법

### 폰/PC 브라우저
`http://<ESP32_IP>/` — 버튼 UI (왼쪽/오른쪽 N바퀴, 연속회전, 정지, 시퀀스)

### API (curl / Claude / 자동화)
```bash
# 예: "왼쪽으로 2바퀴 돌고 1초 쉬고 우측으로 1바퀴"
curl "http://<IP>/api/seq?seq=L2,W1,R1"

curl "http://<IP>/api/turn?dir=left&turns=2"      # 왼쪽 2바퀴
curl "http://<IP>/api/turn?dir=right&turns=1&speed=50"  # 50% 속도
curl "http://<IP>/api/seq?seq=S-100"              # 왼쪽 무한 회전
curl "http://<IP>/api/stop"                       # 정지
curl "http://<IP>/api/status"                     # 상태 JSON
```

시퀀스 문법 (범용 — 조합으로 어떤 응용 명령이든 표현):

| 토큰 | 의미 |
|---|---|
| `L<바퀴>` / `R<바퀴>` | 왼쪽/오른쪽 n바퀴. 소수 가능 (`L0.25` = 왼쪽 90도) |
| `W<초>` | 대기 |
| `S<±속도%>` | 연속회전 (음수=왼쪽), `/api/stop` 까지 |
| `S<±속도%>X<초>` | 시간지정 회전 (`S-50X3` = 왼쪽 50% 로 3초) |
| `@<속도%>` | 해당 스텝 속도 (`L2@50` = 왼쪽 2바퀴를 절반 속도로) |
| `&repeat=N` | 시퀀스 전체 N회 반복 |

응용 명령 변환 예:

| 자연어 명령 | API 호출 |
|---|---|
| 왼쪽 2바퀴 돌고 1초 쉬고 오른쪽 1바퀴 | `seq=L2,W1,R1` |
| 좌우로 반바퀴씩 3번 왔다갔다 | `seq=L0.5,R0.5&repeat=3` |
| 오른쪽으로 천천히 90도 | `seq=R0.25@20` |
| 왼쪽으로 5초간 돌다가 멈추고 오른쪽 3바퀴 | `seq=S-100X5,W0.5,R3` |
| 시계방향으로 계속 돌려 | `seq=S100` |

### 보정 (바퀴 수 정확도)
연속회전 서보는 시간 기반 근사라 개체별 실측 보정 필요:
```bash
curl "http://<IP>/api/cal_run?dir=left"    # 정확히 10초 회전 — 바퀴 수를 눈으로 센다
curl "http://<IP>/api/cal?spr_left=0.625"  # 10초에 16바퀴였으면 10/16=0.625
# 정지 상태에서 미세하게 돌면: curl "http://<IP>/api/cal?stop_us=1480" (1500 근처로 조절)
```
보정값은 NVS 에 저장되어 재부팅 후에도 유지.

## 5. MCP 원격 제어 (AI 자연어 제어 표준 경로)

`mcp-server/` = Rust stdio MCP 서버. ESP32 HTTP API 를 표준 MCP tools 로 노출.
Claude Code 에 user 스코프로 등록 완료 (`claude mcp list` → esp32-motor Connected).

| MCP tool | 기능 |
|---|---|
| `motor_status` | 상태 조회 (회전중/큐/보정값/IP/신호세기) |
| `motor_turn` | 방향+바퀴수+속도 회전 |
| `motor_seq` | 범용 시퀀스 실행 (L/R/W/S 문법 + repeat) |
| `motor_stop` | 즉시 정지 |
| `motor_calibrate` | 보정값 설정 (NVS 저장) |
| `motor_cal_run` | 10초 회전 보정 도우미 |

사용: Claude 세션에서 "모터 왼쪽 2바퀴 돌고 1초 쉬고 오른쪽 1바퀴" → `motor_seq(seq="L2,W1,R1")` 자동 호출.

ESP32 주소 설정 (둘 중 하나):
- 기본: `~/.claude.json` 의 esp32-motor env `ESP32_MOTOR_URL` (현재 AP 폴백 주소 http://192.168.4.1 — 보드가 공유기에 붙으면 실제 IP 로 변경)
- 호출별: 각 tool 의 `host` 인자로 재정의

재빌드: `cd mcp-server && cargo build --release`

MCP 를 안 쓰는 클라이언트(폰 브라우저, curl, 타 자동화)는 4번 API 를 그대로 사용 — 두 경로 모두 ESP32 자체 서버 하나로 수렴.

## 6. 자주 묻는 질문 (완전 초보자용 Q&A)

### 시작 전

**Q. 전자공학을 전혀 몰라도 할 수 있나요?**
네. 납땜이 필요 없습니다. 선 3개를 핀에 꽂고, 프로그램을 USB 로 넣으면 끝입니다. 이 문서 순서대로만 따라 하면 됩니다.

**Q. 부품값은 얼마나 드나요?**
보드+모터+케이블 전부 합쳐 대략 2~3만원 수준입니다 (시세 변동 있음). 컴퓨터는 Windows/Mac 아무거나 됩니다.

**Q. 일반 SG90 서보를 이미 샀는데 쓸 수 있나요?**
각도 제어(0~180도)만 되는 모터라 "2바퀴 돌려" 같은 회전 명령은 불가능합니다. "360도" 또는 "연속회전(continuous rotation)" 표기가 있는 FS90R 계열을 구입하세요. 겉모양은 거의 똑같으니 상품명을 꼭 확인해야 합니다.

### 보드 연결과 드라이버

**Q. 보드를 USB 에 꽂았는데 컴퓨터가 인식을 못 해요.**
1. 케이블 문제가 가장 흔합니다 — "충전 전용" 케이블은 데이터가 안 통합니다. 다른 케이블로 바꿔보세요.
2. 드라이버 미설치일 수 있습니다. 장치관리자(Windows)에서 "알 수 없는 장치"가 보이면 보드 칩에 맞는 드라이버를 설치하세요: CP210x 는 Silicon Labs 홈페이지, CH340 은 WCH 홈페이지에서 무료 배포합니다.
3. USB 허브 말고 컴퓨터 본체 포트에 직접 꽂아보세요.

**Q. ESP32-S3 보드에 USB-C 포트가 2개인데 어디에 꽂나요?**
보드에 작게 "COM"/"UART" 라고 적힌 포트에 꽂으세요. 처음 업로드와 시리얼 모니터는 이 포트가 가장 확실합니다. (다른 하나는 "USB" 포트로, 고급 용도입니다.)

### 펌웨어 업로드

**Q. 업로드하려는데 포트(COM)가 목록에 없어요.**
보드가 인식돼야 포트가 생깁니다. 위의 드라이버 문제부터 해결하세요. Arduino IDE 는 도구 → 포트 메뉴에서 COM 번호를 선택합니다. 보드를 꽂았다 뺐다 하면서 새로 생기는 번호가 내 보드입니다.

**Q. 업로드 중 "Failed to connect" 오류가 나요.**
보드의 **BOOT 버튼을 누른 채로** 업로드를 시작하고, "Connecting..." 문구가 나오면 손을 떼 보세요. 대부분 이걸로 해결됩니다. 그래도 안 되면 케이블/포트를 바꿔보세요.

**Q. 보드 선택을 뭘로 하나요?**
ESP32-S3 보드는 "ESP32S3 Dev Module", 일반 ESP32 보드(칩에 WROOM-32 표기)는 "ESP32 Dev Module" 을 선택하세요. 이 프로젝트 코드는 둘 다 지원합니다.

**Q. 시리얼 모니터에 외계어 같은 글자만 나와요.**
시리얼 모니터 오른쪽 아래 속도(baud rate)를 **115200** 으로 맞추세요.

### 배선 (선 연결)

**Q. 서보 선 3개를 어디에 꽂나요?**
| 서보 선 색 | 보드 핀 | 의미 |
|---|---|---|
| 주황(또는 노랑) | GPIO 4 라고 적힌 핀 | 신호 |
| 빨강 | 5V (또는 VIN) | 전원 |
| 갈색(또는 검정) | GND | 접지 |

핀 이름은 보드 위에 작게 인쇄돼 있습니다. 점퍼케이블 암쪽을 서보 커넥터에, 수쪽을 보드 핀에 꽂으면 됩니다.

**Q. 배선을 잘못 꽂으면 고장 나나요?**
신호선을 다른 GPIO 에 꽂은 정도는 괜찮습니다(동작만 안 함). 단 **빨강(5V)과 갈색(GND)을 서로 바꿔 꽂으면 서보가 고장날 수 있으니** 이 두 개만 주의하세요.

### WiFi 연결

**Q. 우리 집 WiFi 에 연결이 안 돼요.**
ESP32 는 **2.4GHz 전용**입니다. 공유기 이름이 "OOO_5G" 처럼 5GHz 망이면 연결이 안 됩니다. 같은 공유기의 2.4GHz 쪽 이름으로 등록하세요.

**Q. WiFi 비밀번호는 어디에 입력하나요? 코드에 쓰는 건가요?**
아니요, 코드에는 절대 쓰지 마세요 (공개 저장소에 올라가면 비밀번호가 노출됩니다). 두 가지 방법이 있습니다:
- 시리얼 모니터에 `WIFI 공유기이름 비밀번호` 입력 후 엔터 → 자동 저장·재부팅
- 휴대폰 WiFi 목록에서 `ESP32-MOTOR` 에 접속(비밀번호 motor1234) → 브라우저에서 `192.168.4.1` → "WiFi 공유기 등록" 폼 입력

**Q. 비밀번호를 잘못 입력했어요.**
같은 방법으로 다시 입력하면 덮어써집니다. 몇 번이든 다시 하면 됩니다.

**Q. `ESP32-MOTOR` 라는 WiFi 가 폰에 보여요. 이게 뭔가요?**
보드가 공유기 연결에 실패하면 스스로 임시 WiFi 를 만든 것입니다(정상 동작). 여기에 접속해서 `192.168.4.1` 로 들어가면 모터 제어와 공유기 등록을 할 수 있습니다.

**Q. 보드의 IP 주소를 어떻게 알아내나요?**
시리얼 모니터에 `STA connected: ... http://192.168.x.x/` 형태로 출력됩니다. USB 를 이미 뺐다면 공유기 관리 페이지의 "접속 기기 목록"에서 esp32 로 시작하는 기기를 찾으면 됩니다.

**Q. USB 를 빼도 되나요?**
네. WiFi 연결 후에는 USB 는 전원 공급용일 뿐입니다. 휴대폰 충전기나 보조배터리에 꽂아도 똑같이 동작합니다.

### 모터 동작

**Q. 모터가 아예 안 움직여요.**
1. 배선 3개가 표 대로 꽂혀 있는지 확인 (특히 신호선 = GPIO 4)
2. 브라우저에서 `http://<보드IP>/` 접속 → "왼쪽 1바퀴" 버튼을 눌러보세요
3. 그래도 안 되면 시리얼 모니터를 열어 오류 메시지를 확인하세요

**Q. 정지시켰는데도 모터가 조금씩 계속 돌아요.**
연속회전 서보의 개체별 편차입니다(고장 아님). 정지 펄스를 미세조정하면 됩니다:
`curl "http://<보드IP>/api/cal?stop_us=1480"` — 1480, 1520 처럼 1500 근처 값을 바꿔가며 완전히 멈추는 값을 찾으세요. 한 번 찾으면 영구 저장됩니다.

**Q. "2바퀴 돌려" 했는데 2바퀴보다 조금 더/덜 돌아요.**
연속회전 서보는 "시간 기반"이라 처음엔 오차가 있습니다. 보정 한 번이면 크게 좋아집니다:
1. `curl "http://<보드IP>/api/cal_run?dir=left"` → 정확히 10초 돌아갑니다. 몇 바퀴 도는지 세세요.
2. 10 ÷ 바퀴수 를 계산해 등록: 예) 16바퀴였으면 `curl "http://<보드IP>/api/cal?spr_left=0.625"`
3. 오른쪽도 같은 방식(`dir=right`, `spr_right=`)으로 한 번.
그래도 정밀도가 부족하면 스텝모터(28BYJ-48)로 업그레이드가 정답입니다.

**Q. 모터가 덜덜 떨리거나 힘이 없어요.**
USB 포트 전력이 부족한 경우입니다. 벽 충전기(5V 1A 이상)에 꽂아보세요. 서보 여러 개를 달 계획이면 서보 전원은 별도 5V 어댑터로 분리해야 합니다.

### AI / MCP 제어

**Q. MCP 가 뭔가요? 꼭 필요한가요?**
AI(Claude 등)가 외부 도구를 표준 방식으로 쓰게 해주는 규약입니다. 필수는 아닙니다 — 폰 브라우저 버튼이나 curl 만으로도 모든 기능을 쓸 수 있고, MCP 는 "말로 시키기"를 표준화한 추가 경로입니다.

**Q. AI 한테 어떻게 시키나요?**
MCP 등록 후 Claude 에게 평소 말하듯 "모터 왼쪽으로 2바퀴 돌고 1초 쉬고 오른쪽 1바퀴 돌려" 라고 하면 됩니다. AI 가 알아서 `L2,W1,R1` 시퀀스로 바꿔 보냅니다.

**Q. 집 밖에서도 제어할 수 있나요?**
기본 구성은 같은 WiFi(집 안)에서만 됩니다. 외부 제어는 공유기 포트포워딩이나 VPN(Tailscale 등)이 필요한 고급 주제라 이 문서 범위 밖입니다 — 보안 설정 없이 포트를 함부로 열지 마세요.

**Q. 고장 난 것 같아요. 처음부터 다시 하려면요?**
USB 로 펌웨어를 다시 업로드하면 프로그램이 초기화됩니다. 저장된 WiFi/보정값까지 지우려면 Arduino IDE 의 도구 → "Erase All Flash Before Sketch Upload" 를 켜고 업로드하세요.
